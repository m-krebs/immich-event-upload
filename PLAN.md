# immich-event-upload — Implementation Plan

## Overview

A SvelteKit webapp where anyone with a room link can upload photos/videos.
The backend proxies uploads to Immich via its REST API.
Admins (multi-user, authenticated) create rooms; each room auto-creates an Immich album.
A QR code is generated per room for easy sharing.

---

## Stack

- **SvelteKit** (Node adapter, TypeScript)
- **Tailwind CSS v4** — minimal UI
- **better-auth** — email+password, multi-user
- **Drizzle ORM + better-sqlite3** — SQLite
- **qrcode** npm package — server-side SVG generation

## Immich API Client (`src/lib/server/immich.ts`)

Three functions, all reading `IMMICH_URL` and `IMMICH_API_KEY` from env:

```
createAlbum(name: string): Promise<string>
  POST /api/albums  { albumName: name }
  returns albumId

uploadAsset(file: File): Promise<string>
  POST /api/assets  multipart/form-data
  returns assetId

addAssetsToAlbum(albumId: string, assetIds: string[]): Promise<void>
  PUT /api/albums/{albumId}/assets  { ids: assetIds }
```

> Memory/streaming note: by default SvelteKit buffers uploaded files in memory.
> For large video files this is a concern. Revisit when implementing the upload action —
> check whether Immich's API allows the multipart body to be streamed through directly.

## Room Creation Flow (admin)

1. Admin submits room name
2. Server calls `createAlbum(name)` → gets `albumId`
3. Generates a random `token` (e.g. `crypto.randomUUID()` truncated or nanoid)
4. Inserts room row with `immich_album_id = albumId`, `is_active = 1`
5. Redirects to `/admin/rooms/[id]`

---

## Upload Flow (public `/r/[token]`)

1. Load function checks room exists and `is_active === 1`; if not, render closed/not-found state
2. User drags or selects files (images + video — all MIME types Immich supports)
3. Client shows file previews with per-file status indicators (pending / uploading / done / error)
4. On submit: POST all files to the server action
5. Server action:
   - Re-validates room is active
   - Calls `uploadAsset(file)` for all files concurrently (`Promise.all`)
   - Collects resulting asset IDs
   - Calls `addAssetsToAlbum(room.immich_album_id, assetIds)` once
   - Returns per-file success/error map
6. Client updates status indicators per file

**No client-side size limits** — rely on Immich's own validation.
**No compression** — images/video are already compressed; Immich transcodes originals itself.

## Admin Room Detail (`/admin/rooms/[id]`)

- Edit room name
- Toggle `is_active`
- Override `immich_album_id` (text input showing current value)
- Shareable link display + copy button
- QR code rendered as inline SVG (generated server-side via `qrcode`)
- SVG download button
- Delete room button — removes DB record only, Immich album is untouched

---

## Auth & Authorization

- All `/admin/*` routes protected in `+layout.server.ts` via `locals.user`
- Rooms scoped to `created_by` — users only see and manage their own rooms
- `/r/[token]` is fully public, no session required
- No role system needed for now
